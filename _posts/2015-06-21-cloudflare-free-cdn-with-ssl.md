จากคราวก่อนที่มาแนะนำบริการ DNS ฟรีจาก [Cloudflare](https://www.cloudflare.com) ถ้าใครลองไปสมัครเล่นดูแล้วจะพบว่าในหน้าที่ให้เราจัดการ DNS record เนี่ย จะมีปุ่มรูปก้อนเมฆอยู่ต่อท้ายแต่ละ record ด้วย

[![cloudflare-dns-record](//weeix.com/wp-content/uploads/2015/06/cloudflare-dns-record.png)](//weeix.com/wp-content/uploads/2015/06/cloudflare-dns-record.png)

ถ้าปุ่มเป็นสีเทา ก็แสดงว่าเราใช้เพียงแค่บริการ DNS เฉยๆ แต่ถ้าเราลองคลิกเจ้าปุ่มเทานั้นดู ปุ่มจะเปลี่ยนเป็นสีเหลือง! หมายความว่าเราได้เปิดใช้บริการเสริมที่ Cloudflare ได้จัดไว้ให้แล้ว ไม่ว่าจะเป็น CDN, SSL และอื่นๆ

[![cloudflare-dns-record-cdn-enabled](//weeix.com/wp-content/uploads/2015/06/cloudflare-dns-record-cdn-enabled.png)](//weeix.com/wp-content/uploads/2015/06/cloudflare-dns-record-cdn-enabled.png)

ส่วน record ที่ไม่มีปุ่มรูปเมฆต่อท้าย หมายความว่าไม่สามารถเปิดใช้บริการเสริมของ Cloudflare ได้ ตัวอย่าง record เหล่านี้ก็จะมี MX กับ TXT เป็นต้น

[![cloudflare-dns-record-no-cdn](//weeix.com/wp-content/uploads/2015/06/cloudflare-dns-record-no-cdn.png)](//weeix.com/wp-content/uploads/2015/06/cloudflare-dns-record-no-cdn.png)

## CDN

Cloudflare จะมีกลุ่มของ sever ที่ทำหน้าที่เป็น reversed proxy ตั้งอยู่ทั่วโลก อย่างที่ใกล้บ้านเราที่สุดก็น่าจะเป็นกลุ่ม server ในประเทศสิงคโปร์ ทีนี้เวลามีคนเปิดเว็บเรา จากเดิมที่เขาเข้ามาดึงข้อมูลจากเว็บเราโดยตรง ก็เปลี่ยนเป็นไปดึงข้อมูลจาก server ของ Cloudflare ที่อยู่ใกล้ที่สุดแทน ทำให้

*   ประหยัดแบนด์วิธเว็บเรา
*   คนที่เปิดเว็บจากประเทศไกลๆ อย่างอเมริกา จากที่จะต้องโหลดข้อมูลข้ามประเทศ ก็เปลี่ยนไปโหลดจาก Cloudflare ที่อเมริกาแทน ทำให้เปิดเว็บเร็วขึ้นมาก
*   คนไทยที่จะเปิดเว็บของเรา (กรณีเซอร์เวอร์ตั้งอยู่ในไทย) ก็จะต้องไปโหลดข้อมูลจาก Cloudflare ที่สิงคโปร์แทน อันนี้ไม่แน่ใจว่าจะทำให้ช้าลงรึเปล่า

ก็ลองดูละกัน ว่าเราตั้งใจให้ใครเข้ามาใช้เว็บ ถ้าเป็นฝรั่งก็สามารถเปิด CDN ได้โดยแทบไม่ต้องคิดเลย แต่ถ้าเป็นคนไทยก็ชั่งใจดูหน่อย เพราะหัวข้อที่จะพูดถึงต่อไปนี้น่าสนใจมาก

## SSL

หากว่าเราเปิดใช้ CDN แล้ว สิ่งหนึ่งที่ Cloudflare จะแถมให้เราด้วยก็จะเป็นบริการ SSL ฟรี ไม่ว่าเว็บเราจะติดตั้ง SSL Certificate เอาไว้หรือไม่ก็ตาม โดยมีรูปแบบการทำงานต่างๆดังนี้

1.  **Flexible SSL:** สำหรับเว็บที่ไม่สามารถเปิดใช้ https ได้เลย Cloudflare ก็จะเข้ารหัสข้อมูลของเราให้ก่อนส่งไปให้ผู้เข้าชมเว็บ แต่จังหวะที่ Cloudflare มารับข้อมูลจากเราก็ยังมีโอกาสโดนดักจับข้อมูลอยู่นา เพราะฉะนั้นถ้าเลี่ยงได้ก็เลี่ยงนะโหมดนี้ แต่ถ้าเลี่ยงไม่ได้ก็ใช้ไปเถอะ กำขี้ดีกว่ากำตด
2.  **Full SSL:** สำหรับเว็บที่เปิดใช้ https ได้ แต่อาจจะใช้ self-signed certificate หรือ shared certificate เลยทำให้โดนเว็บบราวเซอร์ทั้งหลายกีดกัน เตือนผู้ที่จะมาเข้าชมเว็บไซต์ของเราว่า "certificate ของเว็บนี้ไม่น่าเชื่อถือนะ จะเข้าไปจริงๆเหรอ ถ้าโดนดักจับข้อมูลอย่ามาหาว่ากรูไม่เตือนนะเว้ย" แล้วผู้เข้าชมเว็บเราส่วนมากก็จะกดปิดเว็บเราไปด้วยความหวาดผวา แต่พอเราเปิดใช้ Cloudflare เว็บบราวเซอร์เหล่านั้นก็จะเปลี่ยนท่าที แสดงเครื่องหมายแม่กุญแจสีเขียว ที่หมายความว่าเว็บไซต์นี้ปลอดภัยดี มีการเข้ารหัสข้อมูล ขึ้นมาแทน อันนี้จะดีกว่าแบบแรกคือ ข้อมูลเรามีการเข้ารหัสตั้งแต่จาก server ไปถึง Cloudflare จนกระทั่งไปถึงผู้เข้าชมเว็บ
3.  **Full SSL (strict):** สำหรับเว็บที่ไปซื้อ certificate รายปีมาจากผู้ให้บริการอย่าง Thawte, Symantec, COMODO หรือ GeoTrust มาติดตั้ง ถ้าเลือกตัวเลือกนี้ Cloudflare จะตรวจสอบ certificate เราด้วย ว่าน่าเชื่อถือรึเปล่า ส่วนตัวแล้วไม่แน่ใจจะว่าปลอดภัยกว่าข้อสองรึเปล่า แต่ไหนๆก็จ่ายตังค์ซื้อ certificate ไปแล้ว เลือกๆไปเหอะ 555

อันนี้ไม่แน่ใจว่าตัวระบบจะเลือกให้โดยอัตโนมัติเลยรึเปล่า แต่ถ้าใช้ลองเข้า https://<ชื่อเว็บตัวเอง> แล้วมีปัญหา ก็ลองเข้าไปปรับๆดูในหน้า Crypto นะ
